# Simple Bash Utils

Небольшой проект по разработке собственных версий базовых утилит Bash для работы с текстом: `cat` и `grep`. 

---

## Описание проекта

Программы написаны на языке программирования С (стандарт C11) с использованием компилятора `gcc`. При разработке соблюдались принципы структурного программирования, стиль кода Google Style и стандарт POSIX.1-2017. 

### Реализованный функционал

*   **Утилита cat (`s21_cat`)**: Поддерживает чтение файлов и работу со всеми требуемыми флагами, включая GNU-версии (`-b`, `-e`, `-n`, `-s`, `-t`, `-v`, `-E`, `-T`, `--number-nonblank`, `--number`, `--squeeze-blank`).
*   **Утилита grep (`s21_grep`)**: Осуществляет поиск по шаблонам с использованием библиотеки `regex`. Полностью реализованы базовые флаги (`-e`, `-i`, `-v`, `-c`, `-l`, `-n`), а также дополнительные флаги (`-h`, `-s`, `-f`, `-o`) и их парные комбинации (например, `-iv`, `-in`).
*   **Сборка**: Автоматизирована с помощью `Makefile`, включающего стандартные цели `s21_cat`, `s21_grep`, `clean` и `re`.
*   **Тестирование**: Проект покрыт интеграционными bash-скриптами, которые автоматически проверяют различные комбинации флагов и сравнивают вывод самописных утилит с поведением оригинальных системных команд.

### Особенности тестирования

Проверка реализаций для ОС Alpine проводится с использованием версии 3.20, где утилиты `cat` и `grep` представлены `busybox`-версиями. Следует учитывать, что поведение `busybox` может отличаться от GNU-реализаций.

Для самостоятельного тестирования утилит в среде Alpine 3.20, находясь в корневой папке проекта, можно запустить Docker-контейнер следующей командой:

```bash
docker run -it --rm -v $(pwd):/workspace -w /workspace alpine:3.20 /bin/sh
```

*(Эта команда загрузит образ Alpine 3.20, пробросит текущую папку с твоим кодом внутрь контейнера по пути `/workspace`, перейдет в неё и откроет командную оболочку `sh`. Флаг `--rm` автоматически удалит контейнер после того, как ты из него выйдешь с помощью команды `exit`).*

Внутри контейнера нужно будет установить необходимые инструменты для сборки (компилятор `gcc` и `make`), выполнив команду:

```bash
apk add gcc make musl-dev bash
```
